随笔-Markdown

发表于 2018-09-18 | 分类于 随笔 | | 阅读次数: |

　　折腾了几天，我现在正在对着Markadown GitBook官方文档写下我的个人博客的第一篇博文。

　　配图是我的电脑的桌面图，作为一个图片的链接测试。在编写过程中我发现Markdown并不支持空格，这里再试一下引用语法：

　　「Markdown 语言」不负责实现段首缩进 (ˊ_>ˋ)
　　段首缩进这件事，应该是 CSS 或者其他排版工具的事情，Markdown 奉行的是样式和内容分开的哲学。即使手动输入空格，  也是很不推荐的方法。我推荐全角空格，切换到全角模式下（一般的中文输入法都是按 shift + space）输入两个空格就行了。这个相对   来说稍微干净一点，而且宽度是整整两个汉字，很整齐。

　　这段文字给出了Markdown不支持空格显示的原因以及解决方案。浏览了剩下的部分语法，其实Markdown还是挺简单的。太久没有写作，我发现我已经到了张口无出的地步，当年好歹也还是写作小能手。

　　这两天主要在搭建个人站，踩各种配置的坑，同时开始阅读Go Py 设计模式的书籍，我不知道自己会持续更新多久，但是希望能够坚持写博客以记录每天的琐事，算是每天对自己流逝光阴的负责。今天就先写到这里，后面会专门写一写开设个人站的方法，还有一写技术心得。

　　技术真的太菜了，需要好好努力！
